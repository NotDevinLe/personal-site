1.1T parameters roughly equates to about 550 GB with int4 quantization, 1100 GB with int8 quantization, and so on. Since kimi is natively int4 quantized (and my code expands it to be int8 quantized from the previous blog), I don't have that kind of money, so im going to be using google cloud resources.

Firstly, on google cloud, there is a thing called `buckets` which are like folders but on the cloud and they can store near infinite amounts of data. I store the model weights on this so I don't have to use up all my local pc's memory.

google compute engine allows you to allocate tpus and use them. I'm running a v6e-32 which has 8 workers, each worker having 4 chips, and each chip having 32 gb of ram (HBM). This means that when we shard the model we'll have each chip holding `1100gb / 32w = 34.375gb/w`.
